# 信号判定仕様（STAGE基準）

## 概要

組織診断の各集計結果に対して、改善が必要かどうかを視覚的に判断できる信号（STAGE）を表示する。

## 回答の分類

| 分類 | 回答値 | 意味 |
|-----|-------|------|
| Top2（ポジティブ） | 4 または 5 | 「そう思う」～「強くそう思う」 |
| Mid | 3 | 「どちらとも言えない」 |
| Bottom2（ネガティブ） | 1 または 2 | 「そう思わない」～「全く思わない」 |

## 信号判定基準

| 信号 | 色 | 条件 |
|-----|-----|------|
| **良好** | 青 | 平均得点 ≧ 3.8 **かつ** Bottom2比率 < 10% |
| **注意** | 黄 | 上記どちらか一方のみ満たす（良好でも要改善でもない） |
| **要改善** | 赤 | 平均得点 < 3.8 **かつ** Bottom2比率 ≧ 10% |

### 判定ロジック（疑似コード）

```typescript
function getSignal(score: number, bottom2Rate: number): Signal {
  const scoreOk = score >= 3.8;
  const bottom2Ok = bottom2Rate < 0.10; // 10%未満

  if (scoreOk && bottom2Ok) {
    return "good"; // 青信号：両方OK
  }
  if (!scoreOk && !bottom2Ok) {
    return "bad"; // 赤信号：両方NG
  }
  return "warn"; // 黄信号：片方だけOK
}
```

## 閾値設定

| パラメータ | 値 | 説明 |
|-----------|-----|------|
| SCORE_THRESHOLD | 3.8 | 平均得点の閾値 |
| BOTTOM2_THRESHOLD | 0.10 | Bottom2比率の閾値（10%） |
